# 트릴레마 (Trilemma) — 블록체인과 스테이블코인

## 트릴레마란?

### 철자와 어원

**Trilemma** (트릴레마) — tri(3) + lemma(명제/가정)

- **Dilemma** (딜레마): 2가지 선택지 중 어느 쪽도 좋지 않은 상황
- **Trilemma** (트릴레마): **3가지를 동시에 달성할 수 없는** 상황. 하나를 취하면 다른 하나를 희생해야 함

### 다양한 영역에서의 트릴레마

트릴레마는 블록체인에서 만든 개념이 아니라, **경제학·정치학·철학에서 오래 쓰인 프레임워크**다.

| 영역 | 트릴레마 | 3가지 요소 | 핵심 |
|------|---------|-----------|------|
| **국제경제학** | **먼델의 불가능 삼위일체** (Mundell's Impossible Trinity) | 자유로운 자본 이동 / 독립적 통화정책 / 고정 환율 | 세 가지 중 최대 두 가지만 달성 가능. 한국은 자본 이동+독립 통화정책 선택 → 변동 환율 |
| **프로젝트 관리** | **Iron Triangle** | 빠르게 / 싸게 / 잘 (Fast / Cheap / Good) | "빠르고 좋으면 비싸고, 빠르고 싸면 품질이 낮다" |
| **정치철학** | **아가톤의 트릴레마** | 전지(全知) / 전능(全能) / 전선(全善) | 신이 세 가지를 모두 가졌다면 왜 악이 존재하는가? |
| **블록체인** | **블록체인 트릴레마** | 보안성 / 탈중앙성 / 확장성 | Vitalik Buterin이 명명 |
| **스테이블코인** | **스테이블코인 트릴레마** | 가격 안정성 / 탈중앙성 / 자본 효율성 | 스테이블코인 설계 방식의 근본적 제약 |

**금융인에게 가장 익숙한 것은 먼델의 불가능 삼위일체**다. 이걸 비유로 시작하면 블록체인/스테이블코인 트릴레마를 바로 이해할 수 있다.

## 블록체인 트릴레마 (Blockchain Trilemma)

**명명자:** Vitalik Buterin (이더리움 창시자)

```
            보안성 (Security)
               △
              / \
             /   \
            / BTC  \
           /  ETH   \
          /          \
         / Solana?    \
        /              \
       ◁────────────────▷
  탈중앙성                확장성
  (Decentralization)     (Scalability)
```

### 3가지 요소

| 요소 | 의미 | 측정 |
|------|------|------|
| **보안성** | 네트워크 공격·위변조에 대한 저항력 | 51% 공격 비용, 검증자 수 |
| **탈중앙성** | 소수가 네트워크를 통제할 수 없는 정도 | 노드 수, 진입 장벽, 지분 분산도 |
| **확장성** | 더 많은 트랜잭션을 처리할 수 있는 능력 | TPS, 최종성 시간, 가스비 |

### 각 체인의 선택

| 체인 | 선택 | 희생 | 결과 |
|------|------|------|------|
| **Bitcoin** | 보안+탈중앙 | 확장성 | 7 TPS, 10분 블록 — 가장 안전하지만 가장 느림 |
| **Ethereum** | 보안+탈중앙 | 확장성 | 30 TPS — L2로 확장성 문제 해결 시도 |
| **Solana** | 보안+확장성 | 탈중앙 | 957 TPS — 빠르지만 검증자 하드웨어 요구 높음 (진입 장벽) |
| **BNB Chain** | 확장성+보안(일부) | 탈중앙 | 183 TPS — Binance 21개 검증자가 통제 |

### 왜 동시에 안 되는가

- 탈중앙화하려면 → 많은 노드가 참여해야 → 합의에 시간 걸림 → 느려짐
- 빠르게 하려면 → 참여 노드를 줄이거나 하드웨어 요구를 높이면 → 탈중앙 약화
- 보안을 높이려면 → 검증을 철저히 → 역시 느려짐

### L2가 이 트릴레마의 "치트키"인 이유

- L1은 보안+탈중앙에 집중 (느려도 됨)
- L2가 확장성을 담당 (빠르고 싸게)
- L2의 보안은 L1이 보증 → 트릴레마를 "레이어 분리"로 우회

## 스테이블코인 트릴레마 (Stablecoin Trilemma)

```
          가격 안정성
          (Price Stability)
               △
              / \
             /   \
            / USDC \
           / USDT   \
          /          \
         /    DAI     \
        /              \
       ◁────────────────▷
  탈중앙성            자본 효율성
  (Decentralization)  (Capital Efficiency)
```

### 3가지 요소

| 요소 | 의미 | 예시 |
|------|------|------|
| **가격 안정성** | 항상 $1을 유지하는 능력 | USDC: 거의 완벽 / UST: 붕괴 |
| **탈중앙성** | 특정 주체에 의존하지 않는 정도 | DAI: 높음 / USDC: Circle에 의존 |
| **자본 효율성** | 최소 자본으로 최대 발행할 수 있는 능력 | UST: 담보 0% / DAI: 담보 150% |

### 각 유형의 선택

| 유형 | 선택 | 희생 | 대표 |
|------|------|------|------|
| **법정화폐 담보형** | 안정성 + 자본효율 | **탈중앙** | USDC, USDT |
| | $1 담보 → $1 발행 (효율 100%) | Circle/Tether가 중앙 통제 | |
| | 가격 매우 안정 | 발행사가 동결/블랙리스트 가능 | |
| **암호자산 담보형** | 안정성 + 탈중앙 | **자본효율** | DAI |
| | 스마트 컨트랙트로 탈중앙 운영 | $150 담보 → $100 발행 (효율 67%) | |
| | 과잉 담보로 안정성 확보 | 자본이 묶여서 비효율 | |
| **알고리즘형** | 탈중앙 + 자본효율 | **안정성** | UST (붕괴) |
| | 담보 없이 알고리즘으로 운영 | 담보 0% → 최고 효율 | |
| | 완전 탈중앙화 가능 | **신뢰 붕괴 시 Death Spiral** | |

## 두 트릴레마의 관계

```
블록체인 트릴레마:    어떤 "도로"를 쓸 것인가?
스테이블코인 트릴레마: 그 도로 위에서 어떤 "돈"을 만들 것인가?

두 트릴레마가 겹치는 곳이 실제 설계 결정을 만든다:
- USDC on Ethereum: 블록체인(보안+탈중앙) + 스테이블코인(안정+효율) → 둘 다 중앙화 요소 존재
- DAI on Ethereum: 블록체인(보안+탈중앙) + 스테이블코인(안정+탈중앙) → 자본 비효율
- UST on Terra: 블록체인(보안+확장) + 스테이블코인(탈중앙+효율) → 둘 다 안전성 희생 → 붕괴
```

## 먼델의 불가능 삼위일체와의 비유

금융인에게 설명할 때:

| 먼델 트릴레마 | 스테이블코인 트릴레마 |
|-------------|-------------------|
| 자유로운 자본 이동 | 자본 효율성 |
| 독립적 통화정책 | 탈중앙성 |
| 고정 환율 | 가격 안정성 |

> "한국은 자본 이동의 자유와 독립적 통화정책을 선택하고, 고정 환율을 포기했습니다 (변동 환율제). 스테이블코인도 마찬가지입니다. USDC는 가격 안정성과 자본 효율을 선택하고, 탈중앙성을 포기했습니다. DAI는 안정성과 탈중앙을 선택하고, 자본 효율을 포기했습니다. 그리고 UST는 탈중앙과 효율을 선택하고 안정성을 포기했는데... 그 결과가 $40B 소멸이었습니다."
